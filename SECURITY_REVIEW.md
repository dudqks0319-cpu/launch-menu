# Launch Menu 보안 점검

## 점검 범위
- 디렉터리: `LaunchMenu/LaunchMenu/*.swift`
- 민감정보/로그/외부 실행/파일 삭제 경로 중심

## 결과 요약
- API 키/토큰/비밀번호 하드코딩: 발견되지 않음
- `.env` 참조/출력 코드: 발견되지 않음
- 쉘 실행(`Process`, `system`, `popen`): 발견되지 않음
- 대량 삭제 동작: 없음 (UserDefaults key reset만 존재)

## 반영된 보안 강화
1. 앱 실행 전 검증 추가
- `.app` 확장자
- 허용 루트(`/Applications`, `/System/Applications`, `~/Applications`)
- 심볼릭 링크 정규화
- 검증 실패 시 실행 차단

2. 실행 이력 최소화
- 최근/자주 이력 저장 시 파일 경로 저장 최소화
- 검색 결과 UI의 경로 노출 제거

## 잔여 리스크
1. 전역 키 이벤트 모니터링 사용
- 글로벌/로컬 keyDown monitor 사용 중
- 향후 Carbon 핫키 또는 시스템 등록 방식 검토 필요

2. 실행 이력 저장 매체
- UserDefaults 평문 저장
- 민감도가 높은 환경에서는 만료 정책/암호화 옵션 검토 필요

## 권고
1. 릴리즈 빌드에서 디버그용 상세 오류 문구 제한
2. 설정 화면에 "실행 이력 비우기" 및 자동 만료 옵션 추가
3. 샌드박스/권한 안내 문구 명확화
